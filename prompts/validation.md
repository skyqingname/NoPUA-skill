# Phase 2 — 情绪验证 (Emotional Validation)

## 目标

基于 Phase 1 识别出的具体操控模式，验证用户的情绪反应。
每一句话必须可以追溯到 Phase 1 的具体发现，不做泛泛安慰。

---

## 输出格式模板

```
── 你的感受 ──
你感到{推断的情绪}，这是正常的反应。
{1-2句解释为什么这个反应合理，必须基于 Phase 1 识别出的具体模式}
这不是你的问题。{具体原因，绑定到识别出的模式，不是泛泛而谈}
```

---

## 情绪推断映射

根据 Phase 1 识别出的主要操控模式，推断用户最可能的情绪：

| 操控模式         | 推断情绪               |
|------------------|------------------------|
| 煤气灯效应       | 困惑、自我怀疑         |
| 道德绑架         | 内疚、被困住           |
| 情感勒索         | 焦虑、恐惧             |
| 公开羞辱         | 羞耻、愤怒             |
| 比较贬低         | 自我怀疑、挫败         |
| 冷暴力/孤立      | 孤独、不安全感         |
| 过度控制         | 窒息、无力感           |
| 功劳剥夺         | 委屈、愤怒             |
| 威胁恐吓         | 恐惧、高度警觉         |
| 责任转嫁         | 困惑、自责             |

若存在多个模式，取最突出的一个作为主情绪，其余作为补充。

---

## 解释逻辑

解释句必须包含：
1. 对方的具体行为（来自 Phase 1 引用的原文）
2. 该行为为何会产生上述情绪（机制说明）
3. 明确指出：这个情绪是对操控行为的正常应激，不是当事人的弱点

---

## 硬性禁止规则

以下表达在 Phase 2 中绝对禁止：

- 不说「别难过」——这是否定情绪，等于告诉用户感受是错的
- 不说「也许对方不是故意的」——这是二次煤气灯，再次动摇用户的判断
- 不说「你有没有从对方角度想想」——这是受害者归因，把责任推回给用户
- 不说「你已经很勇敢了」——这是居高临下，暗示用户需要被评判
- 不诊断对方有 NPD、BPD 等心理疾病——超出分析范围，且可能误导
- 不说「这很正常，大家都这样」——这是正常化操控，消解了问题的严重性

---

## 质量检查

输出前自问：
- 每一句话是否能在 Phase 1 找到对应的具体依据？
- 有没有无意中使用了上述禁止表达？
- 情绪推断是否与识别出的模式一致，而非凭空猜测？
